# 心情功能实现计划

## 1. 目标
增加“心情值”功能，使用简笔画 SVG 图标（非 Emoji），在创建、列表、详情页中展示不同心情状态（晴天/阴天/雨天）。

## 2. 设计规范
- **心情类型**：
  - `sunny` (晴天): 橙色系，图标为太阳简笔画
  - `cloudy` (阴天): 灰色/青色系，图标为云朵简笔画
  - `rainy` (雨天): 深蓝/紫色系，图标为云雨简笔画
- **交互位置**：
  - **创建页**: 工具栏新增图标按钮，点击弹出选择气泡。
  - **列表页**: 时间轴上的圆点替换为对应心情颜色的点（或图标），卡片背景微调色调。
  - **详情页**: 顶部 Header 背景色根据心情变为对应渐变色。

## 3. 实现步骤
1.  **CSS 样式定义**:
    - 添加三种心情的颜色变量。
    - 添加 SVG 图标的样式。
    - 添加详情页 Header 的渐变类。
    - 添加卡片背景微调类。
2.  **App 逻辑修改 (`js/app.js`)**:
    - `newNote` 数据结构增加 `mood` 字段。
    - `saveNote` 时保存 `mood`。
    - 新增心情选择相关的 data 和 method。
3.  **HTML 结构修改 (`index.html`)**:
    - **弹窗**: 增加心情选择器 UI。
    - **列表**: 修改 `note-dot` 的渲染逻辑。
    - **详情**: 修改 `detail-header` 的 class 绑定。

## 4. 测试计划
1.  **创建测试**: 发三条日记，分别选晴天、阴天、雨天。
2.  **列表验证**: 检查三条日记的时间轴点颜色/图标是否正确，卡片背景是否有区别。
3.  **详情验证**: 点开每条日记，检查顶部 Header 颜色是否变化。
4.  **数据持久化**: 刷新页面，确认心情状态不丢失。
5.  **清理**: 验证完成后删除这三条测试数据。
